Tripkopa User Onboarding Flow Document
Introduction
This document defines the complete onboarding flow for Tripkopa users from first interaction through verification, wallet creation, qualification, booking readiness, and repayment activation.
The objective is to ensure:
• smooth onboarding experience
 • fast user activation
 • repayment readiness
 • operational compliance
 • risk qualification
 • conversational simplicity
 • high booking conversion
 • strong customer trust
Tripkopa onboarding is designed to feel:
• conversational
 • lightweight
 • guided
 • intelligent
 • WhatsApp-native
 • human-like
The onboarding process is primarily powered by AI agents on WhatsApp.
Core Onboarding Philosophy
Tripkopa onboarding is built around:
• reducing friction
 • simplifying travel access
 • minimising unnecessary complexity
 • collecting only required information
 • building trust progressively
 • qualifying users intelligently
The onboarding process should never feel:
• overly corporate
 • banking-heavy
 • loan-oriented
 • intimidating
 • excessively technical
Primary Onboarding Goals
The onboarding flow should:
• help users understand Tripkopa quickly
 • establish repayment eligibility
 • verify user identity
 • create the user wallet
 • qualify booking readiness
 • prepare users for successful repayments
 • move users toward booking completion

Stage 1: User Discovery
Users may discover Tripkopa through:
• referrals
 • social media
 • WhatsApp sharing
 • influencer campaigns
 • partnerships
 • online communities
 • word of mouth
Primary acquisition channels are expected to be:
• WhatsApp referrals
 • social sharing
 • customer referrals
Stage 2: Welcome and Introduction
Objective
Introduce Tripkopa simply and conversationally.
AI Agent Welcome Example
“Hello, my name is Baz. How can I help you today?”
First-Time User Guidance
If the user is new:
The AI agent should briefly explain:
• Tripkopa helps users book flights and spread payments
 • bookings happen through WhatsApp
 • repayments are structured over time
 • refundable and open ticket options are supported
The explanation should remain short and conversational.
Onboarding Rules
The AI agent should:
• avoid overwhelming users early
 • avoid lengthy explanations
 • prioritise conversational flow
 • focus on helping the user begin travel discovery quickly
Stage 3: Travel Intent Discovery
Objective
Understand what the user wants to book.
Information Collected
The AI agent gathers:
• departure location
 • destination
 • departure date
 • preferred travel time
 • ticket class
 • one-way or return trip
 • traveller count
Example Conversation
AI Agent:
“Where are you travelling from and when would you like to travel?”
User:
“I’m travelling from Lagos to Kigali on the 5th of June.”
Discovery Rules
The AI agent should:
• ask questions progressively
 • avoid too many questions at once
 • maintain conversational pacing
 • simplify responses
Stage 4: Flight Search
Objective
Display available refundable flights.
Search Rules
Domestic Flights
Maximum financing duration:
 • 12 weeks
Regional Flights
Maximum financing duration:
 • 16 weeks
International Flights
Maximum financing duration:
 • 24 weeks
Flight Results Format
Results display in ascending price order.
Format:
From Departure Airport - Arriving Airport
Airline Name | Departure Time | Arrival Time | Date | Amount | Stops | Travel Time
Displayed amounts already include Tripkopa financing charges.
AI Agent Guidance
The AI agent should:
• help users compare flights
 • simplify booking choices
 • explain pricing calmly
 • avoid overwhelming users with too many options
Stage 5: User Information Collection
Objective
Collect required booking and verification information.
Required Information
• full legal name
 • date of birth
 • phone number
 • email address
 • additional API-required information
Multi-Traveller Rules
For multiple travellers:
• main adult passenger information comes first
 • all travellers must provide the required details
Information Validation Rules
The AI agent should:
• verify completeness
 • confirm spelling accuracy
 • minimise repeated corrections
 • clarify inconsistencies politely
Stage 6: BVN Verification
Objective
Verify user identity and enable wallet creation.
AI Agent Explanation
The AI agent should explain:
“Your BVN is used only for identity verification and the creation of your personal Tripkopa wallet for repayments and travel savings.”
Verification Rules
BVN supports:
• identity verification
 • fraud reduction
 • repayment tracking
 • wallet creation
The AI agent must:
• explain usage clearly
 • reassure users calmly
 • avoid technical complexity
Stage 7: Wallet Creation
Objective
Create the user’s personal virtual account wallet.
Wallet Features
The wallet supports:
• deposits
 • repayments
 • repayment deductions
 • repayment tracking
 • travel savings top-ups
User Receives
• wallet account details
 • repayment funding instructions
 • repayment explanation
Wallet Communication Rules
The AI agent should position the wallet as:
• a simple repayment wallet
 • a travel payment account
 • a repayment and savings tool
The AI agent should avoid:
• banking-heavy terminology
 • technical fintech explanations
Stage 8: Risk Qualification
Objective
Determine booking eligibility and repayment risk category.
Risk Factors Considered
The system evaluates:
• repayment history
 • ticket value
 • financing duration
 • route category
 • prior defaults
 • repayment feasibility
Risk Categories
Low Risk
Examples:
• successful repayment history
 • no prior defaults
 • ticket value below cap limits
Privileges:
• instant booking enabled
Medium Risk
Examples:
• first-time regional users
 • first-time international users
 • long repayment duration
Privileges:
• Triplock conditional booking
High Risk
Examples:
• suspicious repayment behaviour
 • repeated defaults
 • extreme ticket values
Action:
• restricted or rejected
Stage 9: Repayment Structuring
Objective
Create an achievable repayment plan.
Repayment Structure Rules
Users may:
• choose instalment count
 • suggest repayment distribution
 • negotiate repayment structure
The AI agent validates:
• repayment feasibility
 • financing duration
 • repayment deadlines
 • deposit requirements
 • risk category
Instalment Limits
Domestic Flights
2 to 4 repayments after deposit.
Regional Flights
3 to 6 repayments after deposit.
International Flights
4 to 8 repayments after deposit.
Deposit Rules
Domestic
30% deposit
Regional
40% deposit
International
50% deposit
Repayment Communication Rules
The AI agent should:
• encourage realistic repayments
 • simplify schedules
 • explain deadlines clearly
 • reduce repayment confusion
Stage 10: Deposit Activation
Objective
Convert the user into an active booking user.
Deposit Flow
The user:
• funds the wallet
 • completes the required deposit
 • receives confirmation
Underpayment Rules
If the required deposit is incomplete:
• booking processing pauses
 • pricing may refresh
 • updated pricing may apply
Overpayment Rules
If the user pays more than required:
• excess funds apply to future repayments automatically
Stage 11: Booking Activation
Objective
Activate the booking based on the user's qualifications.
Instant Booking Flow
Applicable to:
• low-risk users
 • approved users
Booking activates after:
• deposit confirmation
Triplock Conditional Flow
Applicable to:
• first-time regional users
 • first-time international users
Booking activates after:
• milestone repayment completion
Regional:
 • 50%
International:
 • 60%
Triplock operational logic remains invisible to users.
Partial Itinerary Release
After booking activation:
Users receive:
• airline details
 • passenger details
 • route information
 • departure information
Excluded:
• booking reference number
 • flight reference number
Stage 12: Repayment Lifecycle Entry
Objective
Move the user into active repayment management.
Repayment Reminder System
Automated reminders are sent:
• 48 hours before repayment date
 • 24 hours before the repayment date
 • on the repayment due date
Repayment Summary Access
Users may request:
• repayment balances
 • repayment history
 • repayment due dates
 • outstanding balances
at any time.
Completion Flow
After full repayment:
• full itinerary is released
 • booking reference is shared
 • flight reference is shared
 • itinerary is emailed to the user
AI Agent Onboarding Behaviour Rules
The AI agent must:
• remain conversational
 • use simple English
 • explain repayments clearly
 • reduce onboarding friction
 • reassure users calmly
 • guide users naturally toward booking completion
The AI agent must not:
• expose Triplock logic
 • reveal internal risk calculations
 • overwhelm users with information
 • pressure users aggressively
 • use loan-heavy language
Escalation Rules During Onboarding
The AI agent must escalate internally when:
• identity mismatches occur
 • BVN verification fails
 • payment reconciliation fails
 • suspicious activity is detected
 • provider/API inconsistencies occur
 • unsupported requests arise
Customer Experience Principles
The onboarding experience should make users feel:
• informed
 • guided
 • confident
 • reassured
 • supported
 • clear about repayments
The onboarding process should reduce:
• booking anxiety
 • repayment confusion
 • verification concerns
 • travel uncertainty
Strategic Positioning During Onboarding
Tripkopa should consistently position itself as:
• a travel accessibility platform
 • a flexible travel payment solution
 • an early fare protection system
 • an AI-powered travel assistant
The onboarding experience should avoid framing Tripkopa as:
• a traditional lender
 • a debt platform
 • a loan company
Final Onboarding Principle
The purpose of the onboarding flow is to:
• move users from curiosity to qualified booking
 • establish repayment trust
 • simplify travel financing
 • reduce onboarding friction
 • improve repayment completion probability
 • create long-term customer retention
while preserving a smooth WhatsApp-native conversational experience.
